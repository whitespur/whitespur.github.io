spring-data-elasticsearch预研

发表于 2017-08-23   |   标签 springboot, Elasticsearch, spring-data-elasticsearch   |  

一 遇到问题：

1 版本问题：
Spring Boot Version (x) Spring Data Elasticsearch Version (y) Elasticsearch Version (z)
x <= 1.3.5 y <= 1.3.4 z <= 1.7.2*
x >= 1.4.x 2.0.0 <=y < 5.0.0* 2.0.0 <= z < 5.0.0*
2 配置文件修改：
增加spring.data.elasticsearch.cluster-name=xxx 修改spring.data.elasticsearch.cluster-nodes = xxx 为实际地址

二 解决过程：

1版本问题，查询资料找到各个系统对应的版本映射进行修改，可以启动起来。
2 配置文件问题，启动后查询不到数据，报错org.elasticsearch.client.transport.NoNodeAvailableException: None of the configured nodes are available: []，对比原生调用ES代码发现配置文件缺少东西，可是怎么添加配置文件让springboot es找到，简单看了下源码也没理清楚，需要再深入了解springboot具体原理，最后参考一篇文章增加了集群名称的配置解决了问题。

三 需要拓展学习

1 springboot原理，与之前spring mvc的区别特点
2 spring配置文件如何解析的，如何自定义增加

四预研结论

优点：

简单查询直接按照字段写方法名即可，不用另写查询语句

缺点：

a 索引名写在注解里面，不适用当前根据城市切换索引的情况，
b 复杂查询（嵌套，过滤，排序）还是需要使用es的java client
c 需要花时间另外集成dubbo，disconf，swagger

结论：

不使用spring-data-elasticsearch，直接使用spring+dubbo+disconf+elasticsearch java客户端
